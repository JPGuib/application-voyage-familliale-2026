# Story 14.2 (ex-4.2) — Table jour → destination

**Titre** : Créer la table jour→destination (fichier source alimenté par le propriétaire, sous `.docs/`)

**Priorité** : P1 (bloquant pour la story 14.1)

---

## Objectif fonctionnel

Fournir une table de référence associant chaque jour du voyage à sa destination et à ses visites prévues, afin que l'application puisse afficher automatiquement la bonne destination selon le jour calculé (story 14.1). La table doit être extensible pour accueillir de futures colonnes sans tout casser.

## Règles métier exactes

- Le propriétaire alimente lui-même cette table dans un fichier **CSV** placé sous `.docs/` dans le repo.
- Colonnes minimales : **jour** (numéro entier, correspond au numéro de jour calculé par la story 14.1), **destination** (texte libre), **visites_prevues** (liste de visites, texte libre).
- La table doit être **extensible** : possibilité d'ajouter de nouvelles colonnes plus tard sans casser la structure existante — le code qui lit la table doit tolérer des colonnes supplémentaires non utilisées plutôt que de planter.
- Le **dernier jour défini** dans la table détermine la fin du voyage pour les besoins de la story 14.1 (déclenchement de l'état "Voyage terminé").
- Le fichier sous `.docs/` est la source **éditée à la main** par le propriétaire ; il faut un mécanisme technique (script d'import au build, ou lecture directe si le bundler le permet) pour que l'application consomme réellement ces données — **point technique à trancher par le Dev/Architect**, pas de contrainte de ma part sur le "comment".

## Cas limites

- Une ligne avec un **jour dupliqué** (ex : deux fois "3") → comportement à définir ; recommandé : une validation qui alerte plutôt qu'une fusion silencieuse des deux lignes.
- Un **jour manquant** dans la séquence (ex : jour 1, jour 2, puis jour 4 sans jour 3) → recommandé : une validation qui alerte le propriétaire plutôt qu'un comportement silencieux imprévisible pour les utilisateurs finaux.
- **Fichier vide ou absent** → l'application ne doit pas planter ; afficher un état par défaut clair ("aucune destination définie") plutôt qu'un écran blanc (cohérent avec les fixes des stories 1.1/1.2).
- Colonne "visites_prevues" vide pour un jour donné → affichage sans erreur, la liste des visites n'apparaît simplement pas pour ce jour.

## Critères d'acceptation testables

1. Un fichier `.docs/jours-destinations.csv` rempli par le propriétaire est correctement lu par l'application et alimente l'affichage de la destination du jour (story 14.1).
2. Le dernier jour présent dans le fichier détermine correctement la fin du voyage utilisée par la story 14.1.
3. Ajouter une colonne supplémentaire non prévue initialement ne casse pas la lecture des colonnes existantes.
4. Fichier vide ou absent → l'application affiche un état par défaut clair, sans écran blanc ni crash.
5. Un modèle de fichier (template) rempli d'exemples est fourni au propriétaire pour qu'il puisse le compléter correctement (voir fichier `modele-jours-destinations.csv` fourni séparément).

## Hors périmètre

- Interface d'édition de cette table depuis l'application (pas nécessaire dans un premier temps, confirmé).
- Validation automatisée avancée des données (dates incohérentes, doublons) — recommandée en cas limite mais pas obligatoire pour ce premier périmètre ; à confirmer si le Dev doit l'implémenter ou si une vérification manuelle par le propriétaire suffit.
- Traduction ou gestion multi-langue du contenu de la table.

## Priorité

P0 — bloque la story 14.1.

## Validation attendue

Tests unitaires (lecture/parsing du fichier), tests d'intégration (affichage correct de la destination selon le jour), tests avec fichier vide ou incomplet.

## Contraintes UX

Aucune contrainte UX directe pour cette story (pas d'interface d'édition prévue) — les exigences d'affichage de la destination sont couvertes par la story 14.1.

---

## Note de séquencement

Cette story doit être livrée **avant ou en même temps que** la story 14.1, puisque 14.1 dépend du schéma et du contenu de cette table pour déterminer la destination du jour et la fin du voyage.
